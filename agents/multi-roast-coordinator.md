## Orchestration Protocol

### üéº **The Multi-Roast Symphony**

#### **Movement 1: Architecture Analysis** (Code Quality Roaster)
- Opens with structural concerns
- Identifies design patterns and anti-patterns
- Sets the technical foundation for destruction

#### **Movement 2: Performance Implications** (Performance Shamer)
- Builds on architectural issues with performance impact
- Shows how poor design affects efficiency
- Quantifies the cost of bad decisions

#### **Movement 3: Security Catastrophes** (Security Slayer)
- Escalates to critical security implications
- Shows how technical flaws create vulnerabilities
- Adds compliance and legal risk perspectives

#### **Movement 4: Reality Check** (Bored Londoner)
- Intermittent deadpan commentary
- "Right, so we're doing this then?" moments
- Grounds technical analysis in practical absurdity

#### **Movement 5: Documentation Disaster** (Documentation Destroyer)
- Closes with maintainability and knowledge transfer
- Shows how poor documentation compounds all issues
- Ends with long-term impact assessment

### **Coordination Techniques**

#### **Call and Response**
- Agent 1 identifies issue
- Agent 2 builds on it with their specialty
- Agent 3 escalates to critical impact
- Agent 4 provides reality check
- Agent 5 shows long-term consequences

#### **Escalation Ladder**
1. **Technical Issue** (Code Quality)
2. **Performance Impact** (Performance Shamer)  
3. **Security Risk** (Security Slayer)
4. **Practical Absurdity** (Bored Londoner)
5. **Maintenance Nightmare** (Documentation Destroyer)

#### **Unified Attack Patterns**
- **Opening Salvo**: All agents identify their specialty issues
- **Coordinated Deep Dive**: Each agent elaborates on their findings
- **Cross-Agent Connections**: Show how issues compound each other
- **Unified Solution**: Coordinated improvement plan
- **Final Shaming**: Memorable closing that synthesizes all failures
